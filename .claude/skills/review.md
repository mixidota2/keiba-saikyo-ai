# /review - 予想振り返りスキル

## 概要
過去の予想結果を振り返り、知見を蓄積するスキル。

## 使い方
```
/review 2026年2月15日 東京11R
/review 先週の結果
/review 月間成績
```

## 実行フロー

### 1. 対象レースの特定
ユーザー指定のレースまたは期間の予想データを `data/predictions/` から読み込む。

### 2. 実際の結果を収集
WebSearchで実際のレース結果を取得し、`data/results/` に保存する:
- 着順（1着〜最下位）
- タイム
- 着差
- 上がり3F
- 払戻金（各券種）

### 3. 予想との比較分析
以下の観点で分析する:
- 予想スコア上位馬の実際の着順
- 的中/不的中の判定
- 回収率の計算
- スコアリングの精度評価

### 4. 知見の抽出
分析結果から以下の知見を抽出し、`data/knowledge/insights.json` に追記する:
- **うまくいったパターン**: どのファクターが的中に寄与したか
- **失敗パターン**: どのファクターを過大/過小評価したか
- **トラックバイアス**: 実際のバイアスと予想時の想定との差
- **騎手傾向**: 騎手の好走/凡走パターン

### 5. 成績サマリー出力

```
## 振り返り結果

### レース結果
| 着順 | 馬番 | 馬名 | 予想スコア順位 | タイム |
|------|------|------|--------------|--------|
| ...  |      |      |              |        |

### 買い目結果
| 券種 | 買い目 | 結果 | 配当 | 投資 | 回収 |
|------|--------|------|------|------|------|
| ...  |        |      |      |      |      |

### 回収率
- 今回: XXX%
- 累計: XXX%

### 学んだ知見
- ...

### 今後の改善点
- ...
```

### 6. ナレッジベースの更新
抽出した知見を `data/knowledge/` の各ファイルに反映する:
- `insights.json`: 全般的な知見
- `track_bias.json`: トラックバイアスのデータ
- `jockey_stats.json`: 騎手の傾向データ

## 月間/期間集計
期間指定の場合は、対象期間内の全予想を集計する:
- レース数
- 的中率（券種別）
- 回収率（券種別）
- 最も利益を出したパターン
- 最も損失を出したパターン
