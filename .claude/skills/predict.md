# /predict - 競馬レース予想スキル

## 概要
指定されたレースの予想を、Multi-Agentループで実行するメインスキル。

## 使い方
```
/predict 2026年2月15日 東京11R フェブラリーS
/predict 今週の中山メイン
/predict 東京11R 買い目は馬連とワイドのみ
```

### オプション指定
ユーザーが以下のオプションを指定した場合は、ワークフロー全体を通じて厳守する:
- **買い目の券種制約**: 「馬連のみ」「馬連かワイドのみ」「三連複で」など
  → `user_bet_types` として全エージェントに伝達し、最終JSONにも反映する

## 共通ルール
CLAUDE.md の「データ正確性の絶対ルール」「サブエージェントのツール使用ルール」を遵守すること。

## 実行フロー

各エージェントは順番に1つずつ実行する。並列実行はしない。
Critique Agentは毎ループの最終ステップとして必ず実行する。

### Step 1: レース特定と計画立案（Plan Agent）
`agents/plan.md` のプロンプトに従い、Task toolでPlan Agentを起動する。
対象レースの正式情報・収集指示・スコアリング方針を決定する。

### Step 2: 情報収集（Collect Agent）
`agents/collect.md` のプロンプトに従い、Task toolでCollect Agentを起動する。
出走馬・枠順・オッズ・近走成績・持時計・トラックバイアス等を収集し、`data/races/` に保存する。

### Step 3: 推論・スコアリング（Reason Agent）
`agents/reason.md` のプロンプトに従い、Task toolでReason Agentを起動する。
各馬のファクター別スコア算出・期待値計算・推奨買い目を作成する。
ユーザーが券種を指定した場合は `user_bet_types` で指定された券種のみ生成する。

### Step 4: 批評（Critique Agent）
**Reason Agent完了後、必ず実行。スキップ不可。**
`agents/critique.md` のプロンプトに従い、本ループの全出力を検証する。
verdict（sufficient/insufficient）を出力する。

### Step 5: Plan Agentによるループ判断
Critique出力をPlan Agentに渡し、ループ継続/終了を判断させる。
- `insufficient` → 改善計画を作成しStep 2に戻る
- `sufficient` → Step 6へ
- **最大3ループ**。3ループ目完了後はStep 6へ

### Step 6: 最終予想出力

最終予想を以下のフォーマットで出力し、`data/predictions/` に保存する。
ユーザーが券種を指定していた場合、`recommended_bets` には指定券種のみ含める。

```
## 🏇 予想結果: {レース名}

### レース概要
- 日付: YYYY-MM-DD
- 会場: XX
- R: XX
- 距離: XXXX m (芝/ダート)
- 馬場状態: X

### 総合スコアランキング
| 順位 | 馬番 | 馬名 | 総合スコア | 期待回収率 |
|------|------|------|-----------|-----------|
| 1    | XX   | XXX  | XX.X      | XXX%      |
| ...  |      |      |           |           |

### 推奨買い目
- 券種: XXX
- 買い目: XXX
- 推奨投資配分: XXX

### 予想根拠（要約）
- ...

### 注意事項・当日補正ポイント
- 馬体重 ±Xkg以上の変動があれば再評価
- トラックバイアスの確認ポイント
```

## 当日補正について

レース直前にユーザーが追加情報を提供した場合、該当ファクターのスコアを再計算し買い目を修正する。
補正は `/predict` に追加情報を添えて再実行:
```
/predict 補正 東京11R 馬体重: ドウデュース+8kg, トラックバイアス: 内有利
```
