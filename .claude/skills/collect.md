# /collect - 情報収集スキル

## 概要
指定レースの情報のみを収集し、永続化する。予想は行わない。

## 使い方
```
/collect 2026年2月15日 東京11R フェブラリーS
/collect 今週の重賞一覧
```

## 実行フロー

### 1. レース特定
ユーザー指定のレースをWebSearchで特定する。

### 2. 情報収集
以下の情報をWebSearchで収集する:

#### 基本情報
- レース名、日付、会場、距離、コース（芝/ダート）、条件
- 出走馬一覧（馬番、馬名、性齢、斤量、騎手、調教師、馬主）
- 枠順

#### 各馬の詳細情報
各出走馬について以下を収集:
- 近5走の成績（日付、会場、距離、馬場、着順、タイム、着差、上がり3F、騎手、斤量）
- 距離別成績
- コース別成績
- 馬場状態別成績
- 血統情報（父、母、母父）

#### コース・馬場情報
- 当日の馬場状態
- 過去のトラックバイアス傾向（同会場・同条件）
- 天候予報

#### オッズ情報
- 前日オッズ（単勝・複勝）
- 人気順

### 3. データ保存
収集した情報を `data/races/{YYYY-MM-DD}_{会場}_{R}.json` に保存する。

テンプレートは `templates/race_info.json` に従う。

### 4. 収集サマリー出力
収集した情報の概要を表形式で出力する:

```
## 収集完了: {レース名}

### 出走馬一覧
| 馬番 | 馬名 | 騎手 | 前走 | 単勝オッズ |
|------|------|------|------|-----------|
| ...  |      |      |      |           |

### 収集した追加情報
- トラックバイアス: ...
- 馬場状態: ...
- 注目ポイント: ...
```

## 使用するツール
- **WebSearch**: レース情報、出走馬情報、過去成績の検索
- **WebFetch**: 競馬情報サイトからの詳細データ取得
- **Read**: 過去の保存データ参照
- **Write**: 収集データの保存
